# Contribution Guidelines

Thanks for your interest in contributing to this project!

## Code Style

- Use clear and descriptive variable and function names.
- Write comments when the code is not obvious.
- Keep functions short and focused on a single task.

## Pull Requests

- Create a new branch for each feature or bug fix.
- Make sure all changes are committed with clear messages.
- Before opening a pull request, check that the project runs correctly.

## Reporting Issues

- Describe the problem clearly.
- Include steps to reproduce the issue.
- Add screenshots if they help to understand the bug.
